Github Exercise

Target 将项目从本地上传到远程仓库中

  • [X]
    1、在本地建立新的工程，打开Git Bash 输入 git init 生成.git 文件，git status 可查看当前文件状态。
    Tips :
  • 插入的图片可以上传到github中，找到该图片，点击右上角的raw,复制链接即可引用图片。
  • 注意到有untracked files即为未被追踪的档案

2、使用 git add +filename 这个命令来让git追踪

此时在查看文件状态，变成了 changes to commit这种状态叫做stage ,修改过但没使用git add的档案称为unstage

Tips:

  • git add .命令可一次性将所有工作区中的文件加入到暂存区
  • git add -i命令可以在互动模式下选择你要加入的文件

3、使用git commit -m 'message'提交到版本库中

4、在github中new repository。输入关联远程仓库命令 git remote add origin git@github.com:username/repositoryname.git,
git push -u origin master本地项目就上传到了远程仓库中。

Tip:

  • push命令之后出错，如下图：

    原因是新建repository时创建了readme.md文件，而本地文件没有，造成冲突。先git pull origin master,接着重新提交、push即可。
