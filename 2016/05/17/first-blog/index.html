GithubPages建立Blog

Premise

  • Git是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目
  • Node.js是一个开放源代码、跨平台的、可用于服务器端和网络应用的运行环境。Node.js应用JavaScript语言写成，在Node.js运行时运行。
  • Hexo是一个开源的静态博客生成器，用node.js开发

准备工作

  • 下载与安装Git、Node.js
  • 注册github账号,详参 Github配置和使用

GitHubPages

1、使用githubPages创建博客
github建立仓库new a repository 注意仓库名规范 projectname:username.github.io （username即为github的帐户名）
点击create repository。进入下一步选择布局点击publish。
2、地址栏输入username.github.io验证是否可以访问。

Hexo主题

1、选择喜欢的主题，详参Hexo主题排行榜
2、在本地建立文件夹GitBlog在此处右键打开Git Bash
3、安装 Hexo 命令 $ npm install -g hexo
4、部署 Hexo 命令 $ hexo initHexo会自动在该文件夹下建立网站所需要的所有文件。Ps：该文件夹下有_config.yml文件称为站点部署文件
5、复制主题 选择喜欢的主题路径克隆到该文件夹下
$ git clone https://github.com/iissnan/hexo-theme-next themes/next （此处选择的是Next主题）
6、修改站点部署文件 打开_config.yml,找到theme字段进行更改 theme: next (Ps 配置文件中 : 号都要加空格)
7、验证主题 GitBash中输入 $ hexo g,$ hexo s 出现 INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.信息后 ，此时即可使用浏览器访问 http://localhost:4000，检查站点是否正确运行。
8、部署

  • 修改站点部署文件中的URL

    url: http://username.gtihub.io

  • GitBash中输入命令
    $ hexo clean
    $ hexo g
    $ hexo d

  • 可能出现的错误
    1、执行 $ hexo d后出现

    error deployer not found:github
    输入npm install hexo-deployer-git --save
    2、执行 $ hexo d后出现
    erroe failed to execute prompt script

    • 仔细核对github中的repository采用的协议是否与站点配置文件中的一致。即：若github中的协议是ssh ,站点配置文件中的repository修改为ssh协议。
  • 核对github中的repository中的Branch是否为master,如果不是修改为master
  • 是否配置过username、email
    采用该命令$ git config --list查看已设配置，如不存在用户名和邮箱git config --global user.name [username] git config --global user.email [email]
  • 核对完成后重新clean($ hexo clean) 、generate( $ hexo g)、deploy($ hexo d).
